Vision Interface ToolKit

The Vision Interface ToolKit has been created to provide a set of classes and messages that aid in interfacing with environments external to Vision. This ToolKit enables you to standardize your interactions with Vision and provides protocol for arbitrary expression evaluation, structured data extraction, and parameterized queries.

These classes assume that messages defined for built-in classes through release 5.9.4 have been installed. These classes are installed as part of the bootstrap for databases bootstrapped after March 1997.

This interface includes the scripts needed to create the Interface and FormatTools classes. The Vision script SETUP.iface can be used to load the other scripts and assumes that the files are all in your current directory.

Sample C++ Interface Kit

The Vision C++ interface kit consists of a set of C++ classes that you use to provide structured access to Vision data. The classes in this kit provide you with array and character based access to Vision. This kit is designed to interact with the ExtractWS class.

The Vision C++ interface kit is supplied in source code form. You are encouraged to modify that source code to suit your needs.

Click here to transfer these files to your local machine.

See the document, The Vision C++ Interface Kit, for more information.

The VAccess COM Objects

VAccess provides an in-process COM implementation of the capabilities provided by the Vision C++ interface kit. The objects implemented by VAccess use the capabilities of the Vision ExtractWS class to provide you with array and character based access to Vision data.

VAccess is available in both executable and source code form.

See the document, The VAccess COM Objects, for more information.

The VLink OLE Server

VLink is an OLE Automation Server that provides access to Vision data from Visual Basic 3.0, Visual Basic For Applications, and any other language or package capable of using OLE Automation objects or calling a DLL.

This interface is included as part of the Vision Application Client software. Installation of the Vision Application Client software is automated via the install program VisionClient.exe. The install program requires the user to have Administrative Privileges for the current machine. This setup needs connection information related to the Vision Application Server which should be installed and operational prior to the Client setup. This package also includes an Excel add-in that interfaces with the ExtractWS class.

Click here to retrieve the VisionClient installation application. You can put this application anywhere on your local machine, then double click to start the installation. Interactive instructions and notes are provided with the installation. After the Vision Client setup is complete, you can install the MS Excel Add-ins that allow your client to extract information from and upload information to your Vision database.

Be aware that some of the options (Vision Administrator, VUpload.exe) supplied by this installation may not be supported in older Vision installations unless you have installed the appropriate patches.

See the documents The VLink OLE Server and Sample Excel Access for more information about these packages.
